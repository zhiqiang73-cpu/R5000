## ❌ 问题：移动止盈反而让结果变差了！

### 原因分析

#### 1. ATR估算太粗糙
```python
atr_1min = entry_price * Decimal("0.001")  # 固定0.1%
```
**问题**：
- 实际波动率可能远小于0.1%
- 导致移动止盈触发太早
- 盈利很小就启动移动止盈

#### 2. 震荡行情中适得其反
**场景**：
```
入场: 70000
价格涨到70100（盈利0.14%）
→ 移动止盈启动@70050

价格回调到70060
→ 触发移动止盈，只赚60

如果不移动止盈：
→ 价格可能涨到70200，赚200
```

#### 3. 均值回归策略不适合
**特点**：
- 持仓时间短（3分钟）
- 价格快速回归
- 移动止盈来不及发挥作用

## 💡 解决方案

### 方案A：提高移动止盈阈值（保守）

```python
# 从1ATR提升到2ATR
if profit > atr_1min * 2:  # 而不是 atr_1min
    trailing_stop_active = True
```

**效果**：
- 只在大盈利时启动
- 避免小幅震荡触发

### 方案B：只针对趋势跟随（推荐）

```python
# 均值回归不启用移动止盈
if signal.entry_type == "LIMIT":  # 趋势跟随
    # 启用移动止盈
    if profit > atr_1min * 2:
        trailing_stop_active = True
else:  # MARKET（均值回归）
    # 不启用移动止盈
    pass
```

**效果**：
- 均值回归保持原逻辑
- 趋势跟随启用移动止盈
