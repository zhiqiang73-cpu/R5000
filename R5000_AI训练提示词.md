### 第1层：环境过滤

**目的**：只在有优势的环境中交易

**输入**：
- 资金费率（FR）及Z-score
- Open Interest（OI）变化
- 成交量
- 清算流活跃度

**输出**：
- `status`: "可交易" | "休眠"
- `direction_bias`: "偏多" | "偏空" | "中性"
- `liquidation_side`: "多头" | "空头" | None
- `adjustments["stop_multiplier"]`: 0.8-1.3

**关键逻辑**：
```python
# 资金费率极端度
fr_zscore = (fr - fr_mean_7d) / fr_std_7d
if fr_zscore > 2.0:
    direction_bias = "偏空"  # 多头过热
    liquidation_side = "多头"
elif fr_zscore < -2.0:
    direction_bias = "偏多"  # 空头过热
    liquidation_side = "空头"

# OI变化（是否有人在加仓）
oi_change_1h = (oi_now - oi_1h_ago) / oi_1h_ago
if abs(oi_change_1h) > 0.03:  # OI变化超过3%
    activity_level = "高"
```

**验证标准**：
- 休眠时段内强行交易，净亏损占比 > 55%
- 方向偏向与后续清算方向一致率 > 55%

### 第2层：冲击检测 + 分类

**目的**：用清算数据判断是"过度反应"还是"真突破"

**核心创新**：用清算数据作为分类器
```
传统方法（你之前可能见过）：
  用CVD、成交速度、订单簿恢复来判断
  问题：这些都是"症状"，容易被伪造或误判

新方法：
  用"是否触发清算"作为核心判断
  原因：清算是确定性信号——交易所执行的强制平仓，无法伪造
```

**冲击检测**：
```python
# 动态阈值：基于近期波动率
recent_volatility = get_volatility(minutes=60)
impact_threshold = max(0.15%, recent_volatility * 1.5)

# 30秒内价格变动
price_change_30s = abs(price_now - price_30s_ago) / price_30s_ago

# 成交量激增
volume_30s = get_volume(seconds=30)
volume_baseline = get_volume_percentile(75, lookback_windows=60)
volume_surge = volume_30s / volume_baseline

if price_change_30s > impact_threshold and volume_surge > 2.0:
    direction = "up" if price_now > price_30s_ago else "down"
    return True, direction, price_change_30s
```

**分类逻辑（45秒等待窗口）**：
```python
wait_seconds = 45  # 可调参数
time.sleep(wait_seconds)

# 收集清算数据
liquidations = get_liquidations_since(impact_time)

# 计算清算量（与冲击方向一致的）
if impact_direction == "down":
    relevant_liqs = [l for l in liquidations if l.side == "SELL"]  # 多头被强平
else:
    relevant_liqs = [l for l in liquidations if l.side == "BUY"]   # 空头被强平

liq_count = len(relevant_liqs)
liq_value = sum(l.quantity for l in relevant_liqs)
total_volume = get_volume_since(impact_time)
liq_ratio = liq_value / total_volume if total_volume > 0 else 0

# 分类
if liq_count >= 5 and liq_ratio > 0.15:
    return "真突破", {"策略": "趋势跟随", "方向": impact_direction}
elif liq_count <= 2 and liq_ratio < 0.05:
    return "过度反应", {"策略": "均值回归", "方向": "opposite"}
else:
    return "不确定", {"策略": "放弃"}
```

**验证标准**：
- "有清算"的冲击，趋势延续 > 60%
- "无清算"的冲击，价格回归 > 55%
- 分类提升 vs 随机 > 10%

#### 动态杠杆（已实施）

```python
def calculate_position_dynamic_leverage(grade, stop_distance, account_balance, current_price, confidence):
    # 根据信心度动态调整杠杆
    if confidence >= 0.8:
        leverage = 3
        risk_pct = 0.02  # 2%
    elif confidence >= 0.6:
        leverage = 2
        risk_pct = 0.015  # 1.5%
    else:
        leverage = 1
        risk_pct = 0.01  # 1%

    # 根据grade微调
    grade_bonus = {"A": 1.2, "B": 1.0, "C": 0.8}
    risk_pct = risk_pct * grade_bonus[grade]

    risk_usd = account_balance * risk_pct
    qty = risk_usd / stop_distance
    max_qty = (account_balance * leverage * 0.8) / current_price

    return max(0.001, min(qty, max_qty))
```

#### 信号分级

```python
def calculate_grade(confidence, rr_ratio, activity_level):
    score = 0

    # 信心分
    if confidence >= 0.6:  # 注意：>= 不是 >
        score += 2
    elif confidence >= 0.4:
        score += 1

    # 盈亏比分
    if rr_ratio > 1.0:
        score += 1  # 新增：支持低RR均值回归
    if confidence > 0.8: score += 3
    elif confidence > 0.6: score += 2
    if rr_ratio > 2.5: score += 3
    elif rr_ratio > 2.0: score += 2
    elif rr_ratio > 1.5: score += 1

    if activity_level == "高":
        score += 1

    if score >= 6: return "A"  # 风险1.5%
    elif score >= 4: return "B"  # 风险1.0%
    elif score >= 2: return "C"  # 风险0.5%
    else: return "SKIP"
```

#### 熔断规则

```python
class RiskManager:
    def check_circuit_breakers(self):
        # 1. 当日累计亏损 > 3%
        if self.daily_loss > self.total_capital * 0.03:
            return "停止当日交易", "daily_loss_limit"

        # 2. 最近10笔胜率 < 30%
        if len(self.recent_trades) >= 10:
            wins = sum(1 for t in self.recent_trades[-10:] if t.pnl > 0)
            if wins < 3:
                return "暂停1小时", "low_win_rate"

        # 3. 连续3笔亏损
        if len(self.recent_trades) >= 3:
            if all(t.pnl < 0 for t in self.recent_trades[-3:]):
                return "暂停至次日0点", "consecutive_losses"

        # 4. 单笔亏损 > 2%
        if self.recent_trades and self.recent_trades[-1].pnl < -self.total_capital * 0.02:
            return "暂停30分钟 + 人工确认", "large_single_loss"

        return None, None
```

## 🔬 验证方法论（最重要！）

### 核心原则

```
❌ 不做：猜测、直觉、技术指标、复杂模型
✅ 只做：数据验证、逻辑闭环、可重复验证
```

### 验证流程

```
第1步：提出假设
  例如："无清算 = 过度反应 = 价格回归"

第2步：数据验证
  用历史数据统计：
  - 无清算的冲击中，价格回归占比 > 55%？

第3步：回测验证
  - 净期望 > 0（扣手续费后）
  - Sharpe > 0.3
  - 最大回撤 < 5%

第4步：模拟盘验证
  - 模拟盘运行30天
  - 模拟盘 vs 回测偏差 < 20%

第5步：小资金实盘
  - $500-1000起
  - 2x杠杆上限
  - 100笔后评估
```

### 禁止事项

```
❌ 绝对禁止：

1. 不要使用技术指标作为主要信号
   RSI、MACD、均线只能作为辅助
   核心信号是清算数据

2. 不要跳过验证步骤
   每一层必须独立验证通过才能继续
   不通过就停下来修复

3. 不要在回测中使用未来数据
   分类时只能用冲击发生后的数据
   不能用"事后涨跌"作为输入

4. 不要过度优化参数
   参数总数 ≤ 15个
   每个参数必须有逻辑来源

5. 不要忽视手续费和滑点
   所有回测必须扣除真实成本
   成本假设：0.15%双边

6. 不要在趋势中做均值回归
   第1层判断为趋势时，禁用均值回归路径

7. 不要改变已验证模块的接口
   通过验证的模块，接口冻结
   要改就重新验证

8. 不要人工干预自动交易
   系统上线后，人的作用是监控和维护
   不是手动"纠正"系统决策
```

## 🚨 常见陷阱与解决方案

### 陷阱1：移动止盈适得其反

**问题**：
```
实施移动止盈后，Sharpe从0.03降到-1.32
原因：ATR估算太粗糙（固定0.1%）
结果：小幅震荡中频繁止损
```

**解决方案**：
- 先用回测优化参数
- 找到最优ATR倍数
- 再实施到模拟盘

### 陷阱2：PnL计算虚高

**问题**：
```
模拟盘显示+31.61，真实是-31.83
原因：没扣除手续费
```

**解决方案**：
```python
FEE_RATE = Decimal("0.001")  # 0.1%双边
fee = (entry_price * qty + exit_price * qty) * FEE_RATE
pnl = gross_pnl - fee
```

### 陷阱3：样本量不足

**问题**：
```
8笔交易Sharpe 0.04，不够统计显著
```

**解决方案**：
- 至少50笔交易
- 或30天Walk-Forward验证

### 陷阱4：过拟合

**问题**：
```
参数总数 > 15个 → 容易过拟合
```

**解决方案**：
- 每个参数必须有逻辑来源
- Walk-Forward验证参数稳定性

## 🎓 学习要点总结

### 核心思维模式

1. **验证优先**
   - 不猜测，只验证
   - 每次改动都要回测
   - Sharpe不提升就不采用

2. **风险第一**
   - 盈利很重要，但风险控制更重要
   - 最大回撤 < 5%
   - 熔断机制必须可靠

3. **逻辑闭环**
   - 假设 → 验证 → 优化 → 再验证
   - 每一步都可回溯

4. **简洁原则**
   - 参数总数 ≤ 15
   - 每个参数必须有逻辑
   - 代码简洁可读

### 典型工作流

```
1. 发现问题（如：手续费太高）
2. 分析根因（盈利空间 < 手续费）
3. 提出方案（动态杠杆）
4. 实施代码
5. 回测验证
6. 对比效果（Sharpe是否提升？）
7. 决定采用或恢复
8. Git提交并记录
```

### 关键数据

**手续费成本**：
```
Binance Taker: 0.05% * 2 = 0.10%
真实总成本: 0.13-0.17%（含滑点）
```

**盈利目标**：
```
最低: 盈利空间 > 手续费 * 1.5 = 0.15%
目标: 盈利空间 > 0.25%
```

**验证标准**：
```
样本: 至少50笔或30天
Sharpe: > 0.5（考虑实盘）
回撤: < 5%
```
