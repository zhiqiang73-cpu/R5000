# 参数验证规范

## 核心原则

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│  本项目中的所有数字、阈值、百分比都是【占位符】                 │
│                                                                  │
│  它们不是"正确答案"，而是"先这么写，等数据验证"                │
│                                                                  │
│  任何未经真实数据验证的参数，都可能是错的                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

---

## 参数状态标记

每个参数必须有状态标记：

```python
# ❌ 错误：直接写数字，没有说明来源
if liq_count >= 5:
    return "真突破"

# ✅ 正确：标记状态和来源
# [UNVERIFIED] 清算数量阈值
# 来源：猜测
# 待验证：需要采集真实清算数据，分析分布后确定
LIQ_COUNT_THRESHOLD = 5  

if liq_count >= LIQ_COUNT_THRESHOLD:
    return "真突破"
```

### 状态定义

| 标记 | 含义 | 可以上线吗？ |
|------|------|-------------|
| `[UNVERIFIED]` | 未验证，纯猜测 | ❌ 不可以 |
| `[LOGICAL]` | 有逻辑依据，未用数据验证 | ⚠️ 谨慎 |
| `[BACKTESTED]` | 回测验证过 | ⚠️ 谨慎 |
| `[VALIDATED]` | 样本外验证通过 | ✅ 可以 |

## 参数验证流程

### 每个参数都必须经过：

```
┌─────────────────────────────────────────────────────────────────┐
│  第1步：明确假设                                                 │
│  ─────────────                                                  │
│  "我认为清算>=5笔表示真突破"                                    │
│  "为什么是5？因为我猜的"                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第2步：采集数据                                                 │
│  ─────────────                                                  │
│  采集至少7天的清算数据                                          │
│  记录每次冲击后的清算数量                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第3步：分析分布                                                 │
│  ─────────────                                                  │
│  清算数量的均值、中位数、分位数是多少？                         │
│  5笔在分布中处于什么位置？                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第4步：验证假设                                                 │
│  ─────────────                                                  │
│  清算>=5笔时，后续趋势延续率是多少？                            │
│  清算<5笔时，后续趋势延续率是多少？                             │
│  差异显著吗？                                                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第5步：优化阈值                                                 │
│  ─────────────                                                  │
│  测试不同阈值（3、5、8、10、15...）                             │
│  找到区分度最高的点                                              │
│  但不要过拟合！                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第6步：样本外验证                                               │
│  ───────────────                                                │
│  用新的数据验证这个阈值是否仍然有效                             │
│  通过 → 标记为 [VALIDATED]                                      │
│  不通过 → 回到第4步                                             │
└─────────────────────────────────────────────────────────────────┘
```

## 代码中如何标记

### 配置文件方式（推荐）

```python
# config/params.py

class ParamStatus:
    UNVERIFIED = "UNVERIFIED"  # 未验证
    LOGICAL = "LOGICAL"        # 有逻辑依据
    BACKTESTED = "BACKTESTED"  # 回测过
    VALIDATED = "VALIDATED"    # 样本外验证

# 参数定义
PARAMS = {
    "liq_count_threshold": {
        "value": 5,
        "status": ParamStatus.UNVERIFIED,
        "source": "猜测",
        "validation_method": "统计冲击后清算数量分布，验证>=5时趋势延续率",
        "last_validated": None,
        "notes": "核心假设，优先验证"
    },
    
    "impact_threshold_pct": {
        "value": 0.15,
        "status": ParamStatus.UNVERIFIED,
        "source": "猜测",
        "validation_method": "统计30秒价格变动分布，确定异常边界",
        "last_validated": None,
        "notes": ""
    },
    
    "rr_ratio_mean_reversion": {
        "value": 1.3,
        "status": ParamStatus.LOGICAL,
        "source": "手续费覆盖计算：0.15%成本需要盈亏比>1.3",
        "validation_method": "验证真实胜率，重新计算所需盈亏比",
        "last_validated": None,
        "notes": "逻辑正确，但胜率假设需验证"
    },
}

def get_param(name):
    """获取参数值，如果未验证则打印警告"""
    param = PARAMS[name]
    if param["status"] == ParamStatus.UNVERIFIED:
        print(f"⚠️ WARNING: 参数 {name}={param['value']} 未经验证")
    return param["value"]
```

### 使用时

```python
from config.params import get_param

# 使用参数时会自动提醒
threshold = get_param("liq_count_threshold")
# 输出: ⚠️ WARNING: 参数 liq_count_threshold=5 未经验证

if liq_count >= threshold:
    ...
```

## 验证优先级

### 必须首先验证（核心假设）

```
1. 清算数量阈值（真突破 vs 过度反应）
   - 这是整个系统的核心假设
   - 如果这个不成立，其他都没意义

2. 冲击检测阈值
   - 触发器的灵敏度
   - 太高错过机会，太低噪音太多
```

### 其次验证

```
3. 时间相关参数
   - 观察窗口
   - 时间止损
   
4. 风险参数
   - 盈亏比要求
   - 仓位大小
```

### 最后验证

```
5. 辅助参数
   - 流动性阈值
   - FR阈值
   - 止损倍数
```

---

## 禁止事项

```
❌ 禁止：直接使用未验证的参数上实盘

❌ 禁止：因为回测结果好就认为参数正确
   （可能是过拟合）

❌ 禁止：复制其他人的参数
   （市场在变，他们的参数可能已失效）

❌ 禁止：优化出一个"完美"的参数
   （完美 = 过拟合）

❌ 禁止：忽略参数的时间依赖性
   （凌晨的"正常"和下午的"正常"不一样）
```
